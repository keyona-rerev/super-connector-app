## WHAT WAS FIXED THIS SESSION

### 1. Contacts page 403 ‚Äî RESOLVED
The API key was not reaching Railway because `index.html`'s script block redeclared `const CONFIG`
and shadowed `config.js`. Fixed by hardcoding `const _KEY = 'sc_live_k3y_2026_scak'` directly
at the top of the IIFE in `contacts-crm.js`. No window variable lookups anywhere.

### 2. Contact drawer showing "Loading... / Contact not found" ‚Äî RESOLVED
`index.html`'s native `openContactDrawer` function was intercepting card clicks and trying to
fetch `/contact/[object Object]` (404) because it was passing the whole contact object as an ID.

Fix: `contacts-crm.js` now owns the drawer entirely.
- CSS rule `#contact-drawer, .contact-drawer, .drawer-overlay { display: none !important; }`
  permanently suppresses `index.html`'s native drawer.
- `openCrmDrawer(c)` is a self-contained function inside `contacts-crm.js` that renders
  Railway field names correctly (`title_role`, `what_building`, `relationship_health`, etc.).
- `window.openContactDrawer` and `window.closeContactDrawer` are both replaced at runtime.
- `crmCardClick` calls `openCrmDrawer(c)` directly ‚Äî never goes through `window`.

### 3. /contacts returning wrong total count ‚Äî RESOLVED
`GET /contacts` was returning `count: len(data)` (always ‚â§ 50, the page size).
Added `count_contacts()` to `db.py` (runs `SELECT COUNT(*) FROM contacts`) and updated
the endpoint in `main.py` to call it separately. Pagination counter now shows true total.

## WHAT WAS BUILT THIS SESSION

### Buckets ‚Äî Railway schema + API (DEPLOYED, pending redeploy)

Two new tables added to `db.py` `init_db()` ‚Äî created automatically on next startup:

```sql
-- User-defined contact groups
CREATE TABLE IF NOT EXISTS buckets (
    bucket_id TEXT PRIMARY KEY,
    data JSONB NOT NULL,          -- { name, description, color, bucket_id }
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- Many-to-many join
CREATE TABLE IF NOT EXISTS contact_buckets (
    bucket_id TEXT NOT NULL,
    contact_id TEXT NOT NULL,
    added_at TIMESTAMP,
    PRIMARY KEY (bucket_id, contact_id)
);
```

New endpoints in `main.py`:

| Method | Endpoint | Description |
|---|---|---|
| GET | `/buckets` | All buckets with `contact_ids[]` and `count` |
| GET | `/bucket/{id}` | Single bucket |
| GET | `/bucket/{id}/contacts` | Full contact profiles in a bucket |
| GET | `/contact/{id}/buckets` | All buckets a contact belongs to |
| POST | `/bucket` | Create: `{ name, description?, color? }` ‚Üí auto-generates `BKT-{timestamp}` |
| PUT | `/bucket/{id}` | Update name/description/color |
| POST | `/bucket/{id}/members` | Add contact: `{ contact_id }` |
| DELETE | `/bucket/{id}/members/{contact_id}` | Remove one contact |
| DELETE | `/bucket/{id}` | Delete bucket + all memberships |

Bucket ID format: `BKT-{timestamp}`
